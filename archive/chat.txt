ধাপ ৬: Seller/Buyer Filter করা
Python

seller_keywords = ['sell', 'selling', 'wts', 'for sale']
buyer_keywords = ['buy', 'buying', 'wtb', 'looking for']

sellers = []
buyers = []

for msg in messages_data:
    text_lower = msg['text'].lower()
    if any(kw in text_lower for kw in seller_keywords):
        sellers.append(msg)
    elif any(kw in text_lower for kw in buyer_keywords):
        buyers.append(msg)

Private/Secret group-এর invite link ছাড়া join সম্ভব না Secret group সাধারণত invite link বা admin-এর approval ছাড়া join করা যায় না। আপনি যদি link পান এবং join করেন, তাহলে join করার পর থেকের মেসেজ দেখতে পারবেন — কিন্তু join করার আগের পুরনো message history সাধারণত দেখা যায় না (এটা group-এর privacy setting এর উপর নির্ভর করে)।

৩. Admin আপনি না হলে সীমাবদ্ধতা আপনি member হলেও, admin-only কিছু তথ্য (যেমন hidden member list, banned user list) আপনি দেখতে পারবেন না — এই privilege শুধু admin-দের থাকে।

যদি আপনি সেই group-এর সাধারণ member হন (admin না, কিন্তু join করা আছে)
তাহলে scraping সম্ভব — এক্ষেত্রে আগের যেসব script (Telethon দিয়ে iter_messages) বলেছিলাম, সেগুলো কাজ করবে, কারণ member হিসেবে আপনি যা যা দেখতে পান app-এ, script দিয়েও ঠিক সেটাই access করা যায়।

এটা privacy violation এবং অনেক দেশে আইনত অপরাধ (unauthorized data access/hacking-এর আওতায় পড়তে পারে)

১. হ্যাঁ, এগুলো Telegram API-ই ব্যবহার করে
Telethon, Pyrogram — এই লাইব্রেরিগুলো সরাসরি Telegram MTProto API ব্যবহার করে (এটা Bot API না, এটা হলো "User API" / Client API, যেটা দিয়ে normal Telegram app ও কাজ করে)। তাই:

আপনাকে my.telegram.org থেকে api_id ও api_hash নিতে হয়
আপনার নিজের ফোন নম্বর দিয়ে login করতে হয় (OTP verify)
এরপর script আপনার account হিসেবেই Telegram server-এ request পাঠায়
এটা কোনো "third-party hack" না, বরং Telegram নিজেই এই API সবার জন্য open রেখেছে developer app বানানোর জন্য।

২. Rate Limit আছে, কত সেটা fix না — dynamic
Telegram-এর rate limit সরকারিভাবে কোনো fix সংখ্যা প্রকাশ করে না, এটা FloodWait সিস্টেমের মাধ্যমে dynamic ভাবে কাজ করে।

সাধারণ practical limit যা community থেকে জানা যায়:

একসাথে অনেক request পাঠালে Telegram FloodWaitError throw করে, যেখানে নির্দিষ্ট সেকেন্ড (কখনো কখনো কয়েক মিনিট থেকে কয়েক ঘণ্টা পর্যন্ত) wait করতে বলে
নতুন/কম বয়সী account এ limit কড়া, পুরনো verified account এ তুলনামূলক relaxed
একসাথে অনেক group/channel (200+) থেকে scrape করলে account সাময়িকভাবে soft-banned বা restricted হয়ে যেতে পারে
Bypass করার "উপায়" — কিন্তু সতর্কতা সহ
বাস্তবে পুরোপুরি "bypass" বলতে কিছু নেই, তবে যেগুলো মানুষ practice করে:

Technique	কী করে
Sleep/delay যোগ করা	প্রতিটি request-এর মাঝে ২-৫ সেকেন্ড delay দিলে FloodWait কম আসে
Multiple accounts/sessions rotate	একাধিক api_id/session ব্যবহার করে load ভাগ করা
Official Bot API + webhook	scrape এর বদলে bot বানিয়ে group এ add করে event শোনা (এতে flood limit কম প্রযোজ্য)
MTProto Proxy	IP-based ban এড়াতে proxy ব্যবহার, কিন্তু account-level flood wait এতে যায় না
⚠️ কিন্তু এসব bypass করলে Terms of Service violation হতে পারে, এবং worst case-এ আপনার Telegram account permanently ban হতে পারে। Telegram নিজেই এসব bulk-scraping pattern detect করার জন্য অ্যালগরিদম ব্যবহার করে।

৩. হ্যাঁ, Seller/Buyer filter করে scrape করা সম্ভব
এটা করা যায়, তবে সরাসরি Telegram কোনো "role" ট্যাগ (seller/buyer) দেয় না — আপনাকে নিজের লজিক দিয়ে filter করতে হবে। সাধারণত দুইভাবে করা হয়:

পদ্ধতি ১: Keyword-based filter
মেসেজ text-এ নির্দিষ্ট শব্দ (যেমন "selling", "buy", "WTS", "WTB", দাম/price ইত্যাদি) থাকলে সেটাকে seller/buyer হিসেবে ধরা:

Python

for message in client.iter_messages("group_username"):
    if message.text:
        text = message.text.lower()
        if "sell" in text or "wts" in text:
            seller_data.append((message.sender_id, message.text, message.date))
        elif "buy" in text or "wtb" in text:
            buyer_data.append((message.sender_id, message.text, message.date))
পদ্ধতি ২: User ID অনুযায়ী filter (যদি seller/buyer এর ID আগে থেকে জানা থাকে)
Python

known_sellers = [1234567, 7654321]
for message in client.iter_messages("group_username"):
    if message.sender_id in known_sellers:
        print(message.sender_id, message.text)
পদ্ধতি ৩: Reply-chain/thread analyze
Marketplace group-এ সাধারণত buyer, seller-এর মূল post-এ reply করে (message.reply_to_msg_id ফিল্ড চেক করে বোঝা যায় কে কার পোস্টের জবাব দিচ্ছে)।

বাস্তবতা
এই filtering কোনো official Telegram feature না, এটা সম্পূর্ণ আপনার custom script logic-এর উপর নির্ভর করে
Accuracy 100% হবে না, কারণ মানুষ ভিন্নভাবে লেখে ("selling", "want to sell", "up for grabs" ইত্যাদি) — এজন্য অনেকে simple keyword এর বদলে basic NLP/regex ব্যবহার করে
সারকথা: হ্যাঁ এটা Telegram-এর নিজস্ব API দিয়েই হয়, rate limit dynamic (FloodWait ভিত্তিক), সম্পূর্ণ bypass সম্ভব না তবে delay/multiple session দিয়ে কমানো যায়, আর seller/buyer filter করা সম্ভব কিন্তু সেটা নিজের script logic দিয়ে করতে হবে, Telegram নিজে থেকে এই role আলাদা করে দেয় না।

এই স্ক্রিপ্টগুলো ব্যবহার করলে আপনার নিজের Telegram account (api_id/api_hash/phone number দিয়ে login) ব্যবহৃত হয়, তাই misuse করলে আপনার অ্যাকাউন্ট ব্যান হতে পারে।

AI-এর তৈরি করা উত্তরটি আবার টেলিগ্রাম বটের মাধ্যমে ইউজারের কাছে ফেরত পাঠানোর অ্যাকশন সেট করুন। ব্যস, এবার এটি ২৪/৭ মানুষের সাহায্য ছাড়াই চলতে থাকবে।


⚠️ গুরুত্বপূর্ণ সতর্কবার্তা: টেলিগ্রামের নিজস্ব কিছু এপিআই লিমিট (API Rate Limits) আছে। যেমন—এক সেকেন্ডে অনেক বেশি মেসেজ পাঠালে বট সাময়িকভাবে ব্লক হতে পারে। তাই অটোমেশন সেট করার সময় প্রতি মেসেজের মাঝে সামান্য 'Delay' বা বিরতি সেট করে দেওয়া বুদ্ধিমানের কাজ হবে।

আপনি কি কোনো নির্দিষ্ট কাজের জন্য (যেমন: কাস্টমার সাপোর্ট, অটো-পোস্টিং, নাকি ক্রিপ্টো/ট্রেডিং সিগন্যাল) এই অটোমেশনটি করতে চাচ্ছেন? বিস্তারিত জানালে আমি সেই অনুযায়ী সুনির্দিষ্ট গাইডলাইন দিতে পারবো।

Coding free open source tools dekho

কোনো কোডিং ছাড়া (Coding-free) এবং সম্পূর্ণ ওপেন সোর্স (Open-source) টুল ব্যবহার করে টেলিগ্রাম বা যেকোনো বিজনেস ওয়ার্কফ্লো অটোমেট করতে চাইলে বেশ কিছু চমৎকার প্ল্যাটফর্ম রয়েছে। এগুলো আপনি সম্পূর্ণ ফ্রিতে নিজের সার্ভারে বা ডকারে (Docker) হোস্ট করে আনলিমিটেড ব্যবহার করতে পারবেন।

শীর্ষস্থানীয় ৩টি কোডিং-ফ্রি ওপেন সোর্স অটোমেশন টুল নিচে দেওয়া হলো:

১. Activepieces (সবচেয়ে সহজ নো-কোড ইন্টারফেস)
যদি আপনি সম্পূর্ণ ভিজ্যুয়াল এবং ড্র্যাগ-অ্যান্ড-ড্রপ (Drag-and-drop) উপায়ে কাজ করতে চান, তবে এটি সেরা। এর ইন্টারফেস অনেকটাই Make.com বা Zapier-এর মতো সহজ।

টেলিগ্রাম কানেকশন: এটিতে বিল্ট-ইন টেলিগ্রাম পিস (Piece) রয়েছে, যার ফলে কোনো কোড ছাড়াই মেসেজ রিড করা বা রেসপন্স পাঠানো যায়।

এআই মডিউল: OpenAI, Gemini বা local Ollama মডেলগুলোর সাথে কানেক্ট করার জন্য সরাসরি রেডিমেড ব্লক আছে।

লাইসেন্স: MIT/Apicense (ব্যক্তিগত বা ব্যবসায়িক ব্যবহারের জন্য ফ্রিতে স্বাগতিক বা সেলফ-হোস্ট করা যায়)।

GitHub: Activepieces Repository

২. n8n (সবচেয়ে শক্তিশালী ও জনপ্রিয়)
বিশ্বজুড়ে টেকনিক্যাল টিম এবং অটোমেশন এক্সপার্টদের প্রথম পছন্দ n8n। এটি "Fair-code" বা ওপেন-সোর্স মডেলে চলে। এটি অত্যন্ত ফ্লেক্সিবল এবং কন্ডিশনাল রাউটিং (IF/Else লজিক) এর জন্য সেরা।

টেলিগ্রাম কানেকশন: টেলিগ্রাম ট্রিগার (Webhook) এবং টেলিগ্রাম নোড (Node) ব্যবহার করে খুব সহজে মেসেজ, ফাইল বা ইমেজ হ্যান্ডেল করা যায়।

অ্যাডভান্সড এআই: এর নতুন সংস্করণে Advanced AI Nodes যুক্ত করা হয়েছে, যার মাধ্যমে আপনি কোনো কোড ছাড়াই RAG (Knowledge base) এবং এআই এজেন্ট (AI Agent) তৈরি করতে পারবেন।

GitHub: n8n Repository

৩. Kestra (ইভেন্ট-ড্রিভেন এবং ডেক্লারেটিভ)
আপনার যদি বড় ডেটা বা অনেক বেশি ফাইল প্রসেসিংয়ের প্রয়োজন হয়, তবে Kestra একটি চমৎকার অপশন। এটি মূলত YAML (সহজ টেক্সট ফরম্যাট) এবং ভিজ্যুয়াল ইউজার ইন্টারফেস (UI) দুইভাবেই কাজ করে।

টেলিগ্রাম কানেকশন: ১৬০০-এর বেশি প্লাগইনের মধ্যে এর টেলিগ্রাম প্লাগইনটি দিয়ে রিয়েল-টাইম ইভেন্ট হ্যান্ডেল করা যায়।

ডকার সাপোর্ট: ডকার কম্পোজ দিয়ে এক লাইনেই নিজের পিসিতে বা ক্লাউড সার্ভারে চালু করা সম্ভব।

GitHub: Kestra Repository

Coding diye completely free automation open source

যদি আপনি কোডিং করে (Coding-driven) সম্পূর্ণ ফ্রি এবং ওপেন সোর্স উপায়ে টেলিগ্রাম অটোমেশন করতে চান, তবে কোনো থার্ড-পার্টি নো-কোড প্ল্যাটফর্মের (যেমন n8n বা Activepieces) প্রয়োজন নেই। আপনি সরাসরি কোড লিখে স্ক্রিপ্ট বা ব্যাকএন্ড সার্ভিস তৈরি করতে পারেন এবং এটি আপনার লোকাল পিসি, WSL, ডকার বা যেকোনো ফ্রি সার্ভারে ২৪/৭ রান করতে পারেন।

কোডিংয়ের মাধ্যমে সম্পূর্ণ ফ্রিতে টেলিগ্রাম অটোমেশন করার সেরা ওপেন সোর্স ইকোসিস্টেম এবং ফ্রেমওয়ার্কগুলো নিচে দেওয়া হলো:

১. ল্যাঙ্গুয়েজ ও ফ্রেমওয়ার্ক চয়েস (সবচেয়ে জনপ্রিয়)
টেলিগ্রাম বটের জন্য দুটি ল্যাঙ্গুয়েজ সবচেয়ে বেশি ব্যবহৃত হয় এবং এগুলোর ওপেন সোর্স লাইব্রেরিগুলো অত্যন্ত শক্তিশালী:

ক) Python (এআই এবং ডেটা প্রসেসিংয়ের জন্য সেরা)
python-telegram-bot: এটি টেলিগ্রাম এপিআই-এর জন্য সবচেয়ে জনপ্রিয় এবং পরিপক্ক ওপেন সোর্স লাইব্রেরি। এটি সম্পূর্ণ অ্যাসিনক্রোনাস (Asyncio) সাপোর্ট করে, যার ফলে একসাথে হাজার হাজার রিকোয়েস্ট হ্যান্ডেল করা যায়।

Telethon বা Pyrogram: এগুলো মূলত Telegram Client API (MTProto) ব্যবহার করে। এর সুবিধা হলো, সাধারণ বটের যে সীমাবদ্ধতা থাকে (যেমন গ্রুপ থেকে সব মেম্বার স্ক্র্যাপ করা বা কোনো সাধারণ ইউজারের অ্যাকাউন্টকে বট বানানো), তা এই লাইব্রেরিগুলো দিয়ে করা সম্ভব।

GitHub: python-telegram-bot

খ) JavaScript / TypeScript (Node.js)
Telegraf.js: Node.js ইকোসিস্টেমের সবচেয়ে জনপ্রিয় টেলিগ্রাম বট ফ্রেমওয়ার্ক। এটি খুবই লাইটওয়েট এবং ফাস্ট।

GrammY: এটি তুলনামূলক নতুন কিন্তু অত্যন্ত আধুনিক এবং দ্রুতগতির একটি ফ্রেমওয়ার্ক, যা টাইপস্ক্রিপ্ট চমৎকারভাবে সাপোর্ট করে।

GitHub: Telegraf

গ) Rust (হাই-পারফরম্যান্স ও মেমোরি এফিশিয়েন্ট)
Teloxide: আপনি যদি এমন সিস্টেম বানাতে চান যা অত্যন্ত কম র‍্যাম (RAM) এবং সিপিইউ (CPU) ব্যবহার করবে এবং কখনো ক্র্যাশ করবে না, তবে রাস্টের এই ফ্রেমওয়ার্কটি দুর্দান্ত। এটি হাইপার-এফিশিয়েন্ট বট তৈরির জন্য সেরা।

GitHub: Teloxide

২. সম্পূর্ণ ফ্রি আর্কিটেকচার (যেভাবে ফ্রিতে চালাবেন)
কোড লেখার পর সেটি ফ্রিতে এবং মানুষের স্পর্শ ছাড়া (Non-human touch) ২৪/৭ চালানোর জন্য নিচের ওপেন সোর্স টুলস ও আর্কিটেকচার ব্যবহার করতে পারেন:

১. লোকাল এআই ব্রেন (Ollama)
কোনো পেইড এআই এপিআই (যেমন OpenAI) ব্যবহার না করে আপনি যদি সম্পূর্ণ ফ্রি এআই ব্যাকএন্ড চান, তবে Ollama ব্যবহার করতে পারেন।

এটি একটি ওপেন সোর্স টুল যা দিয়ে আপনি আপনার নিজের পিসিতে বা সার্ভারে DeepSeek-R1, Llama 3, বা Mistral-এর মতো শক্তিশালী মডেলগুলো সম্পূর্ণ ফ্রিতে রান করতে পারবেন।

আপনার পাইথন বা নোড কোড সরাসরি এই লোকাল ওলামা এপিআই-এর সাথে কানেক্ট হয়ে চ্যাটের উত্তর জেনারেট করবে।

২. কন্টেইনারাইজেশন (Docker & WSL)
আপনার কোড এবং ওলামা এআই-কে একটি নির্দিষ্ট এনভায়রনমেন্টে ফিক্সড রাখার জন্য Docker ব্যবহার করুন। একটি docker-compose.yml ফাইল তৈরি করে নিলে আপনার পুরো অটোমেশন প্রজেক্টটি এক ক্লিকেই যেকোনো সার্ভারে বা লোকাল মেশিনে ব্যাকগ্রাউন্ডে চলতে থাকবে।

from telegram import Update
from telegram.ext import Application, CommandHandler, MessageHandler, filters, ContextTypes

# স্টার্ট কমান্ড হ্যান্ডলার
async def start(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None:
    await update.message.reply_text('হ্যালো! আমি একটি সম্পূর্ণ স্বয়ংক্রিয় বট।')

# মেসেজ প্রসেসিং এবং অটো-রেসপন্স
async def handle_message(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None:
    user_text = update.message.text
    
    # এখানে আপনি আপনার AI (যেমন ওলামা বা অন্য কোনো ফ্রি API) কল করতে পারেন
    # উদাহরণস্বরূপ একটি সিম্পল লজিক:
    ai_response = f"আপনি বলেছেন: '{user_text}'। আমি এটি স্বয়ংক্রিয়ভাবে প্রসেস করছি..."
    
    await update.message.reply_text(ai_response)

def main():
    # BotFather থেকে পাওয়া টোকেন এখানে বসবে
    app = Application.builder().token("YOUR_TELEGRAM_BOT_TOKEN").build()

    app.add_handler(CommandHandler("start", start))
    app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, handle_message))

    print("বট চালু হয়েছে এবং মেসেজের জন্য অপেক্ষা করছে...")
    app.run_polling()

if __name__ == '__main__':
    main()

১. Python (এআই, অটোমেশন এবং স্ক্র্যাপিংয়ের জন্য সেরা)
পাইথন হলো অটোমেশন দুনিয়ার "সুইস আর্মি নাইফ"। আপনার বটের পেছনে যদি কোনো মেধা বা বুদ্ধিমত্তা (AI) যুক্ত করতে চান, তবে পাইথনই প্রথম পছন্দ।

python-telegram-bot (বট ফ্রেমওয়ার্ক):

কী করা যাবে: অফিসিয়াল কাস্টমার সাপোর্ট বট, অটো-রিপ্লাই সিস্টেম, কোনো ডেটাবেজের (যেমন PostgreSQL/MySQL) সাথে কানেক্ট করে ইউজার ডেটা সেভ করা বা ফর্ম ফিলআপ বট বানানো।

কখন ব্যবহার করবেন: যখন আপনি একটি স্ট্যান্ডার্ড, অফিশিয়াল এবং সিকিউর বট বানাতে চান।

Telethon / Pyrogram (ইউজারবট বা ক্লায়েন্ট ফ্রেমওয়ার্ক):

কী করা যাবে: এটি অফিশিয়াল বটের চেয়ে শক্তিশালী। এটি দিয়ে আপনি নিজের পার্সোনাল টেলিগ্রাম অ্যাকাউন্টকেই ব্যাকগ্রাউন্ডে একটি "বট" বানিয়ে ফেলতে পারবেন (ইউজারবট)। এর মাধ্যমে অন্য কোনো গ্রুপ থেকে স্বয়ংক্রিয়ভাবে মেসেজ বা মিডিয়া ফরওয়ার্ড করা, বড় বড় পাবলিক গ্রুপ থেকে মেম্বারদের ইউজারনেম স্ক্র্যাপ (Scrape) করা, অথবা কোনো গ্রুপে নতুন মেম্বার জয়েন করলে তাকে অটোমেটিক ইনবক্সে মেসেজ দেওয়ার কাজগুলো করা যায়।

কখন ব্যবহার করবেন: যখন সাধারণ বটের এপিআই লিমিটেশন বা বাধাগুলো এড়াতে চান এবং অ্যাকাউন্ট-বেসড স্ক্র্যাপিং বা ক্লোনিং করতে চান।

২. JavaScript / Node.js (রিয়েল-টাইম অ্যাপ ও দ্রুতগতির জন্য সেরা)
ওয়েব ডেভেলপমেন্ট বা এপিআই ইন্টিগ্রেশনের কাজ যদি আপনার বেশি থাকে, তবে Node.js অত্যন্ত দ্রুত কাজ করে।

Telegraf.js / GrammY:

কী করা যাবে: টেলিগ্রামের ভেতরেই মিনি-অ্যাপ বা ওয়েব-অ্যাপ (Telegram Web Apps) তৈরি করা। যেমন—টেলিগ্রাম চ্যাটের ভেতরেই পুরো একটি কাস্টম ইউজার ইন্টারফেস (UI), ড্যাশবোর্ড বা ই-কমার্স শপ খুলে ফেলা। এছাড়া বিভিন্ন থার্ড-পার্টি ওয়েব হুক (যেমন স্ট্রাইপ পেমেন্ট গেটওয়ে, গিটহাব নোটিফিকেশন) সরাসরি টেলিগ্রামে রিয়েল-টাইম পুশ নোটিফিকেশন হিসেবে পাঠানোর জন্য এটি দুর্দান্ত।

কখন ব্যবহার করবেন: যদি আপনার লক্ষ্য হয় টেলিগ্রামের ভেতরে আধুনিক রিচ-টেক্সট, ইনলাইন বাটন এবং কাস্টম মিনি-অ্যাপ ইন্টারফেস তৈরি করা।

কখন ব্যবহার করবেন: যখন সিস্টেমের পারফরম্যান্স এবং মেমোরি ম্যানেজমেন্ট আপনার কাছে সবচেয়ে গুরুত্বপূর্ণ, এবং আপনি চান না যে মেমোরি লিকের কারণে বট কখনো ক্র্যাশ করুক।

গ্রুপ স্ক্র্যাপিং অসম্ভব: অফিশিয়াল বট কোনো পাবলিক বা প্রাইভেট গ্রুপের মেম্বার লিস্ট নিজে থেকে স্ক্র্যাপ (Scrape) বা ডাউনলোড করতে পারে না।

পার্সোনাল অ্যাকাউন্ট চালানো অসম্ভব: এটি দিয়ে আপনি আপনার নিজের সাধারণ টেলিগ্রাম অ্যাকাউন্টকে (যেমন আপনার ব্যক্তিগত ফোন নাম্বার দিয়ে খোলা অ্যাকাউন্ট) ব্যাকগ্রাউন্ডে রোবট বানাতে পারবেন না।

তাহলে এই কাজগুলোর জন্য কী ব্যবহার করতে হবে?
আপনি যে ফিচারগুলোর কথা বললেন (অন্য গ্রুপ থেকে মেসেজ চুরি/ফরওয়ার্ড করা, মেম্বার স্ক্র্যাপ করা, পার্সোনাল অ্যাকাউন্টকে বট বানানো)—এগুলোর জন্য আপনাকে ব্যবহার করতে হবে Telegram Client API (MTProto)।

এর জন্য পাইথনের দুটি নির্দিষ্ট ওপেন সোর্স লাইব্রেরি রয়েছে:

গ্রুপ মেম্বার স্ক্র্যাপিং: যেকোনো বড় পাবলিক গ্রুপের মেম্বার লিস্ট (হাজার হাজার ইউজারের নাম ও ইউজারনেম) এক ক্লিকে স্ক্র্যাপ করে একটি Excel বা CSV ফাইলে সেভ করে ফেলা যায়।

পার্সোনাল অ্যাকাউন্ট অটোমেশন (Userbot): আপনার নিজের আইডি ব্যাকগ্রাউন্ডে একটা স্ক্রিপ্টের মাধ্যমে চলবে। কেউ আপনাকে ইনবক্সে মেসেজ দিলে আপনি ঘুমালেও আপনার আইডি থেকে স্বয়ংক্রিয়ভাবে রিপ্লাই চলে যাবে।

১. Hugging Face Spaces + Docker (সবচেয়ে সেরা ফ্রি সমাধান)
হাগিং ফেস (Hugging Face) মূলত এআই মডেল হোস্ট করার জন্য বিখ্যাত, কিন্তু এটি ১০০% ফ্রিতে ২৪/৭ ব্যাকগ্রাউন্ডে স্ক্রিপ্ট বা বট রান করার জন্য একটি অসাধারণ প্ল্যাটফর্ম।

কোড ছাড়া কীভাবে করবেন: গিটহাবে (GitHub) এমন অনেক রেডিমেড ওপেন সোর্স প্রজেক্ট আছে যেগুলো তৈরিই করা হয়েছে টেলিগ্রাম টু টেলিগ্রাম মেসেজ কপি/ফরওয়ার্ড বা এআই প্রসেসিং করার জন্য (এগুলোকে সাধারণত Telegram Forwarder Bot বা Userbot Clone বলা হয়)।

কাজ করার পদ্ধতি: আপনাকে কোনো কোড লিখতে হবে না। আপনি হাগিং ফেস-এ একটি নতুন Space খুলবেন, সেখানে Docker অপশনটি সিলেক্ট করবেন। এরপর গিটহাবের সেই রেডিমেড প্রজেক্টের লিঙ্কটি দিয়ে "Clone" বা "Mirror" করে দেবেন।

কনফিগারেশন: হাগিং ফেস-এর সেটিংসের ভেতরে Variables বা Secrets নামে একটি জায়গা থাকে। সেখানে আপনার টেলিগ্রামের এপিআই আইডি (API_ID, API_HASH) এবং বট টোকেন বসিয়ে দিলেই হাগিং ফেসের ফ্রি সার্ভারটি ব্যাকগ্রাউন্ডে স্বয়ংক্রিয়ভাবে চালু হয়ে যাবে। এরপর এটি মানুষের কোনো ছোঁয়া ছাড়াই আজীবন চলতে থাকবে।

২. GitHub Actions (টাইম-বেসড বা শিডিউলড অটোমেশন)
গিটহাব অ্যাকশনস (GitHub Actions) মূলত একটি অটোমেশন টুল, তবে এটি ২৪ ঘণ্টা একটানা রান করার জন্য তৈরি হয়নি। এটি তখনই সেরা যখন আপনার কাজটা নির্দিষ্ট সময় পর পর করা দরকার।

কীভাবে কাজ করবে: ধরুন আপনি চান—প্রতি ১ ঘণ্টা পর পর একটি নির্দিষ্ট টেলিগ্রাম চ্যানেল বা গ্রুপ থেকে সব নতুন মেসেজ, ফাইল বা আপডেট স্বয়ংক্রিয়ভাবে স্ক্র্যাপ হয়ে আপনার চ্যানেলে চলে আসবে বা এআই দিয়ে প্রসেস হবে।

কোড ছাড়া ব্যবহার: গিটহাব মার্কেটপ্লেসে (GitHub Marketplace) আগে থেকেই তৈরি করা "Actions Workflow" (রেডিমেড YAML ফাইল) পাওয়া যায়। আপনাকে শুধু একটি ফাইল কপি করে আপনার গিটহাব রিপোজিটরিতে রাখতে হবে এবং একটি ক্রন-জব (Cron Job) বা টাইম সেট করে দিতে হবে (যেমন: */60 * * * * মানে প্রতি ৬০ মিনিট পর পর)। গিটহাব নিজে থেকেই ব্যাকগ্রাউন্ডে আপনার হয়ে কাজটি করে বন্ধ হয়ে যাবে। এটিও সম্পূর্ণ ফ্রি।

n8n.io: এটি একটি ওপেন সোর্স টুল, যা আপনি নিজের সার্ভারে ফ্রিতে হোস্ট করে আনলিমিটেড অটোমেশন চালাতে পারবেন।

Zapier: এটিও বেশ জনপ্রিয়, তবে ফ্রি প্ল্যানে কিছু সীমাবদ্ধতা আছে।

২. এআই (AI) এবং চ্যাটবট ইন্টিগ্রেশন
মানুষের মতো বুদ্ধিমত্তা দিয়ে স্বয়ংক্রিয়ভাবে রিপ্লাই দেওয়ার জন্য আপনাকে বটের সাথে AI যুক্ত করতে হবে।

OpenAI (ChatGPT API) বা Claude API: Make বা n8n-এর মাধ্যমে টেলিগ্রাম বটের সাথে OpenAI API যুক্ত করে দিন। এতে কোনো ইউজার মেসেজ দিলে এআই নিজে থেকেই সেই মেসেজের প্রসঙ্গ বুঝে নিখুঁত উত্তর তৈরি করে পাঠিয়ে দেবে।

ManyChat / Chatfuel: এগুলো মূলত কাস্টমার সাপোর্ট এবং মার্কেটিং অটোমেশনের জন্য ব্যবহৃত হয়। এখানে আগে থেকেই কন্ডিশনাল লজিক সেট করে রাখা যায় (যেমন: ইউজার "Price" লিখলে স্বয়ংক্রিয়ভাবে প্রাইস লিস্ট চলে যাবে)।

Coding free open source tools dekho

কোনো কোডিং ছাড়া (Coding-free) এবং সম্পূর্ণ ওপেন সোর্স (Open-source) টুল ব্যবহার করে টেলিগ্রাম বা যেকোনো বিজনেস ওয়ার্কফ্লো অটোমেট করতে চাইলে বেশ কিছু চমৎকার প্ল্যাটফর্ম রয়েছে। এগুলো আপনি সম্পূর্ণ ফ্রিতে নিজের সার্ভারে বা ডকারে (Docker) হোস্ট করে আনলিমিটেড ব্যবহার করতে পারবেন।

শীর্ষস্থানীয় ৩টি কোডিং-ফ্রি ওপেন সোর্স অটোমেশন টুল নিচে দেওয়া হলো:

১. Activepieces (সবচেয়ে সহজ নো-কোড ইন্টারফেস)
যদি আপনি সম্পূর্ণ ভিজ্যুয়াল এবং ড্র্যাগ-অ্যান্ড-ড্রপ (Drag-and-drop) উপায়ে কাজ করতে চান, তবে এটি সেরা। এর ইন্টারফেস অনেকটাই Make.com বা Zapier-এর মতো সহজ।

টেলিগ্রাম কানেকশন: এটিতে বিল্ট-ইন টেলিগ্রাম পিস (Piece) রয়েছে, যার ফলে কোনো কোড ছাড়াই মেসেজ রিড করা বা রেসপন্স পাঠানো যায়।

এআই মডিউল: OpenAI, Gemini বা local Ollama মডেলগুলোর সাথে কানেক্ট করার জন্য সরাসরি রেডিমেড ব্লক আছে।

লাইসেন্স: MIT/Apicense (ব্যক্তিগত বা ব্যবসায়িক ব্যবহারের জন্য ফ্রিতে স্বাগতিক বা সেলফ-হোস্ট করা যায়)।

GitHub: Activepieces Repository

২. n8n (সবচেয়ে শক্তিশালী ও জনপ্রিয়)
বিশ্বজুড়ে টেকনিক্যাল টিম এবং অটোমেশন এক্সপার্টদের প্রথম পছন্দ n8n। এটি "Fair-code" বা ওপেন-সোর্স মডেলে চলে। এটি অত্যন্ত ফ্লেক্সিবল এবং কন্ডিশনাল রাউটিং (IF/Else লজিক) এর জন্য সেরা।

টেলিগ্রাম কানেকশন: টেলিগ্রাম ট্রিগার (Webhook) এবং টেলিগ্রাম নোড (Node) ব্যবহার করে খুব সহজে মেসেজ, ফাইল বা ইমেজ হ্যান্ডেল করা যায়।

অ্যাডভান্সড এআই: এর নতুন সংস্করণে Advanced AI Nodes যুক্ত করা হয়েছে, যার মাধ্যমে আপনি কোনো কোড ছাড়াই RAG (Knowledge base) এবং এআই এজেন্ট (AI Agent) তৈরি করতে পারবেন।

GitHub: n8n Repository

৩. Kestra (ইভেন্ট-ড্রিভেন এবং ডেক্লারেটিভ)
আপনার যদি বড় ডেটা বা অনেক বেশি ফাইল প্রসেসিংয়ের প্রয়োজন হয়, তবে Kestra একটি চমৎকার অপশন। এটি মূলত YAML (সহজ টেক্সট ফরম্যাট) এবং ভিজ্যুয়াল ইউজার ইন্টারফেস (UI) দুইভাবেই কাজ করে।

টেলিগ্রাম কানেকশন: ১৬০০-এর বেশি প্লাগইনের মধ্যে এর টেলিগ্রাম প্লাগইনটি দিয়ে রিয়েল-টাইম ইভেন্ট হ্যান্ডেল করা যায়।

ডকার সাপোর্ট: ডকার কম্পোজ দিয়ে এক লাইনেই নিজের পিসিতে বা ক্লাউড সার্ভারে চালু করা সম্ভব।

নিচে একটি বেসিক এপিআই-বেসড বটের কোড দেওয়া হলো যা মানুষের ছোঁয়া ছাড়াই মেসেজ রিসিভ করে স্বয়ংক্রিয়ভাবে প্রসেস করতে পারে:

from telegram import Update
from telegram.ext import Application, CommandHandler, MessageHandler, filters, ContextTypes

# স্টার্ট কমান্ড হ্যান্ডলার
async def start(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None:
    await update.message.reply_text('হ্যালো! আমি একটি সম্পূর্ণ স্বয়ংক্রিয় বট।')

# মেসেজ প্রসেসিং এবং অটো-রেসপন্স
async def handle_message(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None:
    user_text = update.message.text
    
    # এখানে আপনি আপনার AI (যেমন ওলামা বা অন্য কোনো ফ্রি API) কল করতে পারেন
    # উদাহরণস্বরূপ একটি সিম্পল লজিক:
    ai_response = f"আপনি বলেছেন: '{user_text}'। আমি এটি স্বয়ংক্রিয়ভাবে প্রসেস করছি..."
    
    await update.message.reply_text(ai_response)

def main():
    # BotFather থেকে পাওয়া টোকেন এখানে বসবে
    app = Application.builder().token("YOUR_TELEGRAM_BOT_TOKEN").build()

    app.add_handler(CommandHandler("start", start))
    app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, handle_message))

    print("বট চালু হয়েছে এবং মেসেজের জন্য অপেক্ষা করছে...")
    app.run_polling()

if __name__ == '__main__':
    main()

কী করা যাবে: এমন একটি বট যা একসাথে লাখ লাখ ইউজারের মেসেজ প্রসেস করবে কিন্তু আপনার সার্ভারের মাত্র ১০-২০ মেগাবাইট র‍্যাম (RAM) খরচ করবে। এটি দিয়ে হাই-স্পিড ডেটা প্রসেসিং, ট্রেডিং বটের অ্যালগরিদম রান করা, বা চ্যাটে আসা বড় বড় ফাইল ব্যাকগ্রাউন্ডে খুব দ্রুত কোনো ক্লাউড স্টোরেজে আপলোড করার মতো জটিল পাইপলাইন তৈরি করা যায়।

কখন ব্যবহার করবেন: যখন সিস্টেমের পারফরম্যান্স এবং মেমোরি ম্যানেজমেন্ট আপনার কাছে সবচেয়ে গুরুত্বপূর্ণ, এবং আপনি চান না যে মেমোরি লিকের কারণে বট কখনো ক্র্যাশ করুক।

৪. Ollama (সম্পূর্ণ ফ্রি এবং লোকাল এআই ব্রেন)
আপনার কোডটি যদি বটের "হাত-পা" হয়, তবে ওলামা হলো তার "মাথা"।

Ollama (Llama 3, DeepSeek-R1, Mistral):

কী করা যাবে: এই এআই মডেলগুলোকে আপনার পাইথন বা নোড কোডের সাথে কানেক্ট করে সম্পূর্ণ "হিউম্যান-লেস" চ্যাটিং করা যাবে। ইউজার যেকোনো প্রশ্ন করলে এআই নিজেই কনটেক্সট বুঝে মানুষের মতো উত্তর লিখে দেবে। কোনো কাস্টমার যদি প্রোডাক্টের দাম বা সাপোর্ট চায়, এআই আপনার দেওয়া ডেটা ফাইল (Knowledge Base) পড়ে সঠিক তথ্যটি বের করে রিপ্লাই দেবে। এর জন্য ওপেন-এআই বা অন্য কাউকে ১ টাকাও দিতে হবে না।

সংক্ষেপে সিদ্ধান্ত নেওয়ার গাইড:
আপনার যদি AI Agent, চ্যাটবট, ডেটা স্ক্র্যাপিং বা মেসেজ ফরওয়ার্ডিং দরকার হয় ➔ Python (Pyrogram / python-telegram-bot) + Ollama বেছে নিন।

আপনার যদি টেলিগ্রাম মিনি-অ্যাপ, কাস্টম UI বা রিয়েল-টাইম এপিআই নোটিফিকেশন দরকার হয় ➔ Node.js (GrammY / Telegraf.js) বেছে নিন।

আপনার যদি আল্ট্রা-ফাস্ট, ক্র্যাশ-ফ্রি এবং ২৪/৭ লার্জ-স্কেল ডাটা প্রসেসিং দরকার হয় ➔ Rust (Teloxide) বেছে নিন।

আপনি মূলত কোন আইডিয়া বা ফিচারটি সবার আগে ইমপ্লিমেন্ট করতে চাচ্ছেন? (যেমন: কোনো নির্দিষ্ট গ্রুপ থেকে মেসেজ কপি করা, নাকি এআই দিয়ে কাস্টমার সাপোর্ট দেওয়া?)

python-telegram-bot (বট ফ্রেমওয়ার্ক): eta diye eta kora jabe naa?কী করা যাবে: এটি অফিশিয়াল বটের চেয়ে শক্তিশালী। এটি দিয়ে আপনি নিজের পার্সোনাল টেলিগ্রাম অ্যাকাউন্টকেই ব্যাকগ্রাউন্ডে একটি "বট" বানিয়ে ফেলতে পারবেন (ইউজারবট)। এর মাধ্যমে অন্য কোনো গ্রুপ থেকে স্বয়ংক্রিয়ভাবে মেসেজ বা মিডিয়া ফরওয়ার্ড করা, বড় বড় পাবলিক গ্রুপ থেকে মেম্বারদের ইউজারনেম স্ক্র্যাপ (Scrape) করা, অথবা কোনো গ্রুপে ন

না, আপনি যেটা ভাবছেন—python-telegram-bot দিয়ে এই কাজগুলো (ইউজারবট, অন্য গ্রুপ থেকে মেসেজ স্ক্র্যাপ বা ফরওয়ার্ড করা) সরাসরি করা যাবে না। এখানেই অফিশিয়াল বট আর ইউজারবটের মধ্যে মূল পার্থক্য। সহজ ভাষায় বুঝিয়ে বলছি কেন যাবে না এবং এর সমাধান কী:

কেন python-telegram-bot দিয়ে এটা সম্ভব নয়?
python-telegram-bot ফ্রেমওয়ার্কটি তৈরি করা হয়েছে টেলিগ্রামের Bot API-এর ওপর ভিত্তি করে (যেগুলোর ইউজারনেমের শেষে _bot লেখা থাকে, যেমন: @MyAutomated_bot)। অফিশিয়াল বটের কিছু বড় সীমাবদ্ধতা বা নিয়মের বেড়াজাল থাকে:

অন্য গ্রুপের মেসেজ দেখতে না পাওয়া: একটি অফিশিয়াল বটকে যদি আপনি অন্য কোনো গ্রুপে অ্যাড করেন, তবে সে গোপনীয়তার কারণে (Privacy Mode) অন্য সাধারণ মেম্বারদের মেসেজ রিড করতে পারে না—যতক্ষণ না তাকে ওই গ্রুপের 'Admin' বানানো হচ্ছে।

গ্রুপ স্ক্র্যাপিং অসম্ভব: অফিশিয়াল বট কোনো পাবলিক বা প্রাইভেট গ্রুপের মেম্বার লিস্ট নিজে থেকে স্ক্র্যাপ (Scrape) বা ডাউনলোড করতে পারে না।

পার্সোনাল অ্যাকাউন্ট চালানো অসম্ভব: এটি দিয়ে আপনি আপনার নিজের সাধারণ টেলিগ্রাম অ্যাকাউন্টকে (যেমন আপনার ব্যক্তিগত ফোন নাম্বার দিয়ে খোলা অ্যাকাউন্ট) ব্যাকগ্রাউন্ডে রোবট বানাতে পারবেন না।

তাহলে এই কাজগুলোর জন্য কী ব্যবহার করতে হবে?
আপনি যে ফিচারগুলোর কথা বললেন (অন্য গ্রুপ থেকে মেসেজ চুরি/ফরওয়ার্ড করা, মেম্বার স্ক্র্যাপ করা, পার্সোনাল অ্যাকাউন্টকে বট বানানো)—এগুলোর জন্য আপনাকে ব্যবহার করতে হবে Telegram Client API (MTProto)।

এর জন্য পাইথনের দুটি নির্দিষ্ট ওপেন সোর্স লাইব্রেরি রয়েছে:
১. Telethon ২. Pyrogram

এই দুটি (Telethon/Pyrogram) দিয়ে ঠিক কী কী করা যাবে?
মেসেজ ও মিডিয়া অটো-ফরওয়ার্ড: আপনি যেকোনো থার্ড-পার্টি পাবলিক বা প্রাইভেট গ্রুপ/চ্যানেলে (যেখানে আপনি একজন সাধারণ মেম্বার হিসেবে জয়েন আছেন) নজর রাখতে পারবেন। সেখানে নতুন কোনো মেসেজ বা ফাইল (যেমন: সিগন্যাল, ছবি, পিডিএফ) আসামাত্রই আপনার কোড সেটি কপি করে আপনার নিজের চ্যানেল বা গ্রুপে সেকেন্ডের মধ্যে ফরওয়ার্ড করে দেবে।

গ্রুপ মেম্বার স্ক্র্যাপিং: যেকোনো বড় পাবলিক গ্রুপের মেম্বার লিস্ট (হাজার হাজার ইউজারের নাম ও ইউজারনেম) এক ক্লিকে স্ক্র্যাপ করে একটি Excel বা CSV ফাইলে সেভ করে ফেলা যায়।

পার্সোনাল অ্যাকাউন্ট অটোমেশন (Userbot): আপনার নিজের আইডি ব্যাকগ্রাউন্ডে একটা স্ক্রিপ্টের মাধ্যমে চলবে। কেউ আপনাকে ইনবক্সে মেসেজ দিলে আপনি ঘুমালেও আপনার আইডি থেকে স্বয়ংক্রিয়ভাবে রিপ্লাই চলে যাবে।

বৈশিষ্ট্য	python-telegram-bot (Bot API)	Pyrogram / Telethon (Client API)

অন্য গ্রুপের মেসেজ রিড	গ্রুপ অ্যাডমিন না হলে অসম্ভব।	আপনি সাধারণ মেম্বার হিসেবে জয়েন থাকলেই সব মেসেজ রিড করা যাবে।
মেম্বার স্ক্র্যাপিং	সম্পূর্ণ নিষিদ্ধ/অসম্ভব।	খুব সহজেই মেম্বার লিস্ট স্ক্র্যাপ করা যায়।
ব্যান হওয়ার ঝুঁকি	একদম নেই (টেলিগ্রাম অনুমোদিত)।	একটু বেশি (খুব দ্রুত বা স্প্যামিংয়ের মতো মেসেজ ফরওয়ার্ড করলে পার্সোনাল অ্যাকাউন্ট ব্যান হতে পারে)।
সংক্ষেপে: আপনি যদি অন্য কোনো গ্রুপ বা চ্যানেল থেকে ডেটা স্ক্র্যাপ করতে চান বা মেসেজ অটো-ফরওয়ার্ড করতে চান, তবে python-telegram-bot বাদ দিয়ে সরাসরি Pyrogram অথবা Telethon দিয়ে কোড লেখা শুরু করুন।

আপনি কি এই ধরণের কোনো স্ক্র্যাপিং বা মেসেজ ফরওয়ার্ডিং প্রজেক্ট তৈরি করতে চাচ্ছেন? তাহলে আমি আপনাকে Pyrogram-এর একটি বেসিক কোড স্ট্রাকচার দেখিয়ে দিতে পারি।

১. Telethon ২. Pyrogram duita alada project? konta valo? duto same kaaj kore? egulo ki opensourse github project naa?

হ্যাঁ, আপনার ধারণা একদম সঠিক—এগুলো দুটোই ওপেন সোর্স GitHub প্রজেক্ট। এগুলো সম্পূর্ণ ফ্রি এবং এদের সোর্স কোড গিটহাবে উন্মুক্ত।

আপনার প্রশ্নগুলোর সহজ এবং বিস্তারিত উত্তর নিচে দেওয়া হলো:

১. এগুলো কি আলাদা প্রজেক্ট? দুটো কি একই কাজ করে?
হ্যাঁ, এগুলো সম্পূর্ণ আলাদা দুটি প্রজেক্ট, তবে এদের কাজ মূলত একই। দুটোই টেলিগ্রামের MTProto API (Client API) ব্যবহার করে। অর্থাৎ, দুটো লাইব্রেরি দিয়েই আপনি আপনার পার্সোনাল অ্যাকাউন্টকে অটোমেট করা, অন্য গ্রুপ থেকে মেসেজ ফরওয়ার্ড করা বা মেম্বার স্ক্র্যাপ করার কাজগুলো করতে পারবেন।

সহজ কথায়, গন্তব্য এক, কিন্তু যাওয়ার রাস্তা দুটো আলাদা।

২. Pyrogram বনাম Telethon: কোনটি ভালো?
দুটোই চমৎকার, তবে কোডিং স্টাইল এবং আপডেটের দিক থেকে এদের মধ্যে কিছু বড় পার্থক্য রয়েছে:

Pyrogram (আধুনিক এবং সহজ)
এটি বর্তমানে পাইথন ডেভেলপারদের মধ্যে সবচেয়ে জনপ্রিয়। এর কোড লেখা তুলনামূলক সহজ এবং পরিষ্কার।

কেন ভালো: এটি সম্পূর্ণ অ্যাসিনক্রোনাস (Asyncio) এবং এর ডকুমেন্টেশন খুব সুন্দর। নতুনদের জন্য Pyrogram দিয়ে কোড করা অনেক সহজ।

GitHub লিংক: Pyrogram Repository

Telethon (পুরনো এবং অত্যন্ত নির্ভরযোগ্য)
এটি টেলিগ্রামের অন্যতম প্রাচীন এবং পরিপক্ক (Mature) পাইথন লাইব্রেরি।

কেন ভালো: দীর্ঘদিন ধরে বাজারে থাকায় এর কমিউনিটি অনেক বড়। অনেক জটিল এবং অ্যাডভান্সড কাজের জন্য Telethon-এর কার্যক্ষমতা দারুণ। তবে এর কোডিং স্টাইল নতুনদের কাছে কিছুটা কঠিন মনে হতে পারে।

GitHub লিংক: Telethon Repository

১. Hugging Face Spaces + Docker (সবচেয়ে সেরা ফ্রি সমাধান)
হাগিং ফেস (Hugging Face) মূলত এআই মডেল হোস্ট করার জন্য বিখ্যাত, কিন্তু এটি ১০০% ফ্রিতে ২৪/৭ ব্যাকগ্রাউন্ডে স্ক্রিপ্ট বা বট রান করার জন্য একটি অসাধারণ প্ল্যাটফর্ম।

কোড ছাড়া কীভাবে করবেন: গিটহাবে (GitHub) এমন অনেক রেডিমেড ওপেন সোর্স প্রজেক্ট আছে যেগুলো তৈরিই করা হয়েছে টেলিগ্রাম টু টেলিগ্রাম মেসেজ কপি/ফরওয়ার্ড বা এআই প্রসেসিং করার জন্য (এগুলোকে সাধারণত Telegram Forwarder Bot বা Userbot Clone বলা হয়)।

কাজ করার পদ্ধতি: আপনাকে কোনো কোড লিখতে হবে না। আপনি হাগিং ফেস-এ একটি নতুন Space খুলবেন, সেখানে Docker অপশনটি সিলেক্ট করবেন। এরপর গিটহাবের সেই রেডিমেড প্রজেক্টের লিঙ্কটি দিয়ে "Clone" বা "Mirror" করে দেবেন।

কনফিগারেশন: হাগিং ফেস-এর সেটিংসের ভেতরে Variables বা Secrets নামে একটি জায়গা থাকে। সেখানে আপনার টেলিগ্রামের এপিআই আইডি (API_ID, API_HASH) এবং বট টোকেন বসিয়ে দিলেই হাগিং ফেসের ফ্রি সার্ভারটি ব্যাকগ্রাউন্ডে স্বয়ংক্রিয়ভাবে চালু হয়ে যাবে। এরপর এটি মানুষের কোনো ছোঁয়া ছাড়াই আজীবন চলতে থাকবে।

২. GitHub Actions (টাইম-বেসড বা শিডিউলড অটোমেশন)
গিটহাব অ্যাকশনস (GitHub Actions) মূলত একটি অটোমেশন টুল, তবে এটি ২৪ ঘণ্টা একটানা রান করার জন্য তৈরি হয়নি। এটি তখনই সেরা যখন আপনার কাজটা নির্দিষ্ট সময় পর পর করা দরকার।

কীভাবে কাজ করবে: ধরুন আপনি চান—প্রতি ১ ঘণ্টা পর পর একটি নির্দিষ্ট টেলিগ্রাম চ্যানেল বা গ্রুপ থেকে সব নতুন মেসেজ, ফাইল বা আপডেট স্বয়ংক্রিয়ভাবে স্ক্র্যাপ হয়ে আপনার চ্যানেলে চলে আসবে বা এআই দিয়ে প্রসেস হবে।

কোড ছাড়া ব্যবহার: গিটহাব মার্কেটপ্লেসে (GitHub Marketplace) আগে থেকেই তৈরি করা "Actions Workflow" (রেডিমেড YAML ফাইল) পাওয়া যায়। আপনাকে শুধু একটি ফাইল কপি করে আপনার গিটহাব রিপোজিটরিতে রাখতে হবে এবং একটি ক্রন-জব (Cron Job) বা টাইম সেট করে দিতে হবে (যেমন: */60 * * * * মানে প্রতি ৬০ মিনিট পর পর)। গিটহাব নিজে থেকেই ব্যাকগ্রাউন্ডে আপনার হয়ে কাজটি করে বন্ধ হয়ে যাবে। এটিও সম্পূর্ণ ফ্রি।

This logs in as your actual phone number account, not a bot.
